We frequently discuss highly optimized Design Systems, open-source UI kits, and sophisticated 
libraries like shadcn/ui or Material UI.

However, in practice, projects operate under very different constraints: an in-house design standard, 
multiple contributing teams, evolving business requirements, and increasingly, the rapid arrival of 
AI-assisted development. There is no simple “plug-and-play” solution that keeps everything aligned.

This talk shares my hands-on experience maintaining an internal React component library in this 
evolving environment: how to establish clear standards without stifling innovation, how to accelerate 
development using AI as a practical assistant rather than a magic solution, and how to keep the 
library stable, understandable, and genuinely useful for all teams.

Rather than focusing on idealized architectures, this presentation provides a grounded perspective 
on the real challenges encountered in the field. It explores how AI can support documentation, 
testing, and refactoring workflows while teams still navigate organizational constraints, 
differing expectations, and the constant need for pragmatic compromises.

Disclaimer: I will not dive into React-specific implementation details, though some 
API and code design patterns may be discussed.
