Dans de plus en plus de projets nous avons besoin d'utiliser des librairies pour partager du code et des fonctionnalités, le tout avec des versions différentes de frameworks. Avec CamelStudio, nous avons rencontré ce besoin sur un de nos projets de développer une librairie Java qui soit compatible avec Java 17 mais aussi avec Java 8 :scream:
Pour répondre à ce besoin nous avons utilisé la solution des artefacts multi-profils permettant de construire des versions spécifiques pour chaque environnement cible et avec la mise en place d'une Gitlab CI pour tester et générer facilement les différents artefacts.
Dans cette conférence je présenterai comment fonctionnent les artefacts multi-profils et comment les mettre en place.
Au programme :
- Introduction sur les artefacts Java
- Génération d'un librairie pour Java 17
- Introduction sur le classifier et les artefacts multi-profils
- Génération de la lib avec deux profils pour une compatibilité Java 17 et Java 8
- Mise en place Gitlab CI pour release, package et versionning des artefacts
